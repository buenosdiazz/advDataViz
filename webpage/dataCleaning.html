Working with Data

In this week’s class, we discussed data cleaning and different ways in which we can work with raw data.

Some of the main takeaways were:

  • Begin with data in plain text formal (if possible)
  • Do not perform calculations in raw data file
  • Avoid spaces (use dashes and underscores instead)
  • Use lowercase
  • Fill in all cells
During class, we also took a look at the Titanic data set and examined how we could improve it with data cleaning. This data set provided information about various characteristics about passengers on the Titanic, such as passenger gender, class, age, and whether or not the person survived.

After reviewing the data, we noted that there were several improvements that could be made.
  • Separate the name variable into last name, first name, and title variables
  • Use “1,2,3” instead of “1st, 2nd, 3rd” for the pclass variable
  • Separate home.dest variable into two variables “city” and “state”
  • Fill empty cells with “null”
Using excel we also discovered some interesting information by filtering with variables and using the COUNTIF function. For example, we learned that more women survived than men and first class had the highest survival rate.

We also discussed the extra considerations that are needed when using data in code. Some JavaScript friendly variable types are:
  • String
  • Number
  • Boolean (true/false)
  • Arrays
  • Objects
Whenever we are using data with code it is important to conform to the programming language’s syntax.

Coincidentally, the first time I heard about data cleaning was last week at my job at the Toppel Career Center. My supervisor, the Associate Director of Assessment & Communication, is in charge of collecting the data from the graduating student survey that provides a snapshot of students’ post-graduation plans. She held a “data-cleaning” party in which everyone in the office got together to review and revise the survey data.

Practicing data cleaning in my job/ seeing its real-world application helped me understand how much work it is (it takes a village)! Even though it seems tedious, it is necessary to make sure that the results produced from the data are consistent and accurate.
